# Code Review Agent (Phase 1 MVP)

An AI-powered Python code reviewer. Paste or upload Python code and receive
structured feedback (issues, severity, suggestions, and an overall summary)
generated by an LLM through the Groq API.

## 1. Project Overview

The Code Review Agent accepts Python source code and returns an automated review:

- **Issues Found** with a **Severity** label (High / Medium / Low)
- **Suggestions** for improvement
- An **Overall Summary** of the code's quality

It is a clean two-tier MVP built with a Streamlit frontend, a FastAPI backend,
and Groq (Qwen3-32B) as the LLM provider.

## 2. Architecture

The system has a strict separation between presentation and logic:

- **Streamlit (frontend)** handles all UI concerns: input, file upload,
  loading states, and rendering results. It contains no business logic.
- **FastAPI (backend)** exposes a REST API, validates input with Pydantic,
  and orchestrates the review.
- **Reviewer module** is the only component that talks to Groq. It builds the
  prompt, calls Qwen3-32B asynchronously, and parses the JSON response.
- **Config module** centralizes environment configuration and validation.

### Data Flow

1. User pastes code or uploads a `.py` file in Streamlit.
2. Streamlit `POST`s `{ "code": "..." }` to FastAPI `/review`.
3. FastAPI validates the body (`ReviewRequest`, including size limits).
4. The reviewer sends a structured prompt to Groq (Qwen3-32B), requesting JSON.
5. Groq returns JSON, which is safely parsed into a `ReviewResponse`.
6. FastAPI returns the validated response; Streamlit renders it.

## 4. Setup Instructions

1. Clone the repository and enter it:
   ```bash
   git clone <your-repo-url>
   cd code-review-agent
   ```
2. Create and activate a virtual environment:
   ```bash
   python -m venv .venv
   source .venv/bin/activate        # Windows: .venv\Scripts\activate
   ```
3. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```
4. Create your `.env` file from the template and add your Groq API key:
   ```bash
   cp .env.example .env
   # Edit .env and set GROQ_API_KEY
   ```

## 5. Environment Variables

| Variable          | Required | Default            | Description                                  |
|-------------------|----------|--------------------|----------------------------------------------|
| `GROQ_API_KEY`    | Yes      | ‚Äî                  | Your Groq API key.                           |
| `GROQ_MODEL`      | No       | `qwen/qwen3-32b`   | Groq chat model id.                          |
| `REQUEST_TIMEOUT` | No       | `60`               | Timeout (seconds) for the Groq API call.     |
| `MAX_CODE_CHARS`  | No       | `50000`            | Maximum allowed length of submitted code.    |
| `BACKEND_URL`     | No       | `http://localhost:8000` | Backend URL used by the Streamlit frontend. |

## 6. Running the Backend

From the project root:

```bash
cd backend
uvicorn main:app --reload --port 8000
```

The API is now available at `http://localhost:8000`. Interactive docs
(Swagger UI) are at `http://localhost:8000/docs`.

## 7. Running the Frontend

In a second terminal (with the virtual environment activated):

```bash
streamlit run frontend/streamlit_app.py
```

Open the URL Streamlit prints (usually `http://localhost:8501`).

## 8. Testing the API

Check the health endpoint:

```bash
curl http://localhost:8000/health
# {"status":"ok"}
```

Send a review request:

```bash
curl -X POST http://localhost:8000/review \
  -H "Content-Type: application/json" \
  -d '{"code": "def add(a,b): return a+b"}'
```

### API Endpoints

| Method | Path      | Description                          |
|--------|-----------|--------------------------------------|
| `GET`  | `/health` | Health check. Returns `{"status":"ok"}`. |
| `POST` | `/review` | Reviews Python code, returns structured feedback. |

**Error responses**

- `422` ‚Äî invalid input (empty or exceeds `MAX_CODE_CHARS`).
- `502` ‚Äî upstream LLM failure or unparseable model output.

## 9. Example Request

`POST /review`

```json
{
  "code": "def add(a,b): return a+b"
}
```
